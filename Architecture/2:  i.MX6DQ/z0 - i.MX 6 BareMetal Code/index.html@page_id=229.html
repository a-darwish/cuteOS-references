Getting Started With ARM i.MX6Q Bare Metal on Mac OS X

Before You Start

You need:

• An i.MX6Q board (e.g. BD-SL-i.MX6 Freescale SABRE Lite compatible device.

• A USB to RS232 cable (e.g. DSE XH8290).

• A micro-USB to USB cable

• g++ for ARM (e.g from MacPorts: “port install arm-none-eabi-gcc”)

• minicom (e.g. form MacPorts: “port install minicom”)

• IMX6Q Bare Metal Getting Started Source Code including makefile or each file separately:

  • imx_run.c
  • imx6q_uart.c and the linker script imx6q.ld and the assembly language start-up imx6q.s

Setup

• Compile imx_run and also compile imx6q_uart and imx6q.s with imx6q.ld. either “make” (if you have the zip file) or:

c++  -x c++ -Wno-dangling-else -o imx_run.mac imx_run.c -framework CoreFoundation -framework IOKit
arm-none-eabi-gcc -mcpu=cortex-a9 -DIMX6Q -DSABRE_LITE -g3 -Wall -Os -l gcc -ffreestanding -nostartfiles -o imx6q_uart.elf imx6q.s imx6q_uart.c  -T imx6q.ld

• Connect the USB to RS232 with the USB end on the Mac and the RS232 connected to the serial port on the SABRE Lite board. I use the “CONSOLE” connector which is UART2.

• Connect the micro-USB end of the USB cable to the SABRE Lite and the USB end to the Mac

• Power up the SABRE Lite board. The light should come on.

• Fund out which tty port is acting as the serial port. In my case its /dev/tty.usbserial-FTG4ND95. The easiest way to do this is to disconnect the FTDI cable, ls /dev/tty* then connect the FTDI cable and see what has changed.

Bare Metal

• start minicom

minicom -D /dev/tty.usbserial-FTG4ND95

set minicom to “115200 8N1″ as the baud, bits, party, and stop. This is done (within minicom) using “Ctrl-A Z P”

• upload and run imx6q_uart.elf

./imx_run.mac imx6q_uart.elf

• Check the minicom window. I get “Compiled on Aug 23 2013 at 15:12:22″
