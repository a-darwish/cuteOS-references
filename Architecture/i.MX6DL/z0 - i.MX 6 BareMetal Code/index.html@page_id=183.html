i.MX6Q Programming Development Environment

In order to compile and test your i.MX6Q Bare Metal program you need a development environment. I’ll assume you have a source code editor on your machine. The remainder is described here.

C/C++ Compiler

Mentor Graphics have a build of The GNU tool chain that is free for Windows and Linux. They call it “Sourcery CodeBench Lite Edition”. You want the EABI version. It’s really easy to download and install.

For MacOS its slightly different. You need a package manager and the compiler. I used MacPorts as it has (nearly) the latest build.
1. Install http://www.macports.org
2. Tell MacPorts to get gcc

sudo port install arm-none-eabi-gcc

3. verify that the install was successful

arm-none-eabi-gcc -v

Terminal Emulator

For MacOS I’m using minicom. It can be installed from MacPorts

sudo port install minicom

You now need to create a serial port. I use a generic USB to RS232 cable (FTDI cable) I got cheap from the local Radio Shack equivalent. Remember to install any device drivers that might come with it (or find the on the FTDI web site). Windows (until recently) came with a terminal emulator. I use the same USB to RS232 cable and software form the FTDI web site as the device driver.

On the Boundary Devices board I’m using I’ve got the FTDI cable attached to the serial port labeled “CONSOLE” which is actually UART2!

Upload Software

You need software to upload an executable from the host development platform to the i.MX6Q board. As I want to boot the i.MX6Q over the USB I wrote my own, imx_run. It looks for an i.MX6Q device attached as a HID over USB and then uses the Serial Download Protocol (See section 8.8 (page 452) of “i.MX 6Dual/6Quad Applications Processor Reference Manual Rev. 1, 04/2013″ for details of the protocol).
